# About 페이지 디자인 재구성 계획

## 목표
- 새 `AGENTS.md`와 `DESIGN.md`를 기준으로 About 페이지를 dark-first `founder war-room / blueprint desk` 스타일로 재구성한다.
- 예전 green/light 스타일은 유지하지 않는다.
- 현재 콘텐츠의 의미와 핵심 정보는 유지하되, 문구 톤은 더 전술적이고 정밀한 소개 페이지처럼 조정한다.
- `src/style.css`는 `@import "tailwindcss";`만 남긴다.
- 디자인 표현은 Tailwind v4 유틸리티 클래스를 TSX 마크업에 직접 작성한다.

## 추천 접근
- `src/About/About.tsx`는 단순 조립 컴포넌트에서 “planning desk” 레이아웃 컨테이너로 확장한다.
- 하위 섹션 컴포넌트는 기본적으로 유지하되, 디자인 일관성을 위해 필요한 class/마크업은 일부 수정한다.
- `src/style.css`는 Tailwind import 한 줄만 남겨 CSS 엔트리 역할만 맡긴다.
- `DESIGN.md`의 시각 언어는 `planner-bg` 같은 커스텀 CSS 클래스가 아니라 Tailwind 유틸리티 조합으로 구현한다.
- 테스트는 CSS 문자열 검증을 줄이고 HTML 구조, 핵심 콘텐츠, 동작 계약 중심으로 최소 보정한다.

## 수정 대상 파일
- `src/About/About.tsx`
  - `main.planner-bg.about-page.min-h-screen` 구조로 변경
  - `about-shell`, `war-room-grid`, `command-rail`, `workspace-stack` 같은 상위 레이아웃 래퍼 추가
  - 섹션들을 좌/우 작업면처럼 재배치
- `src/style.css`
  - `@import "tailwindcss";` 한 줄만 유지
  - 기존 전역 CSS는 모두 제거
- `src/About/*.tsx`
  - 필요한 경우 일부 섹션의 class명/래퍼를 새 디자인 primitives에 맞게 보정
  - `hero-preview`, `site-footer`, copy 관련 `data-copy-target`, `id`, inline script hook은 유지
- `src/index.test.ts`
  - 기존 CSS 문자열 고정 검증을 새 디자인 핵심 클래스 검증으로 보정
  - copy 동작 계약, 핵심 콘텐츠, 메타 태그 검증은 유지

## About.tsx 구조

- 비대칭 planning grid
  - 왼쪽: `HeroSection`, attribution/status rail
  - 오른쪽: definition, capabilities, install, command usage, quick links
- 모바일은 단일 컬럼으로 자연스럽게 쌓이도록 설계

## style.css 구성
- `@import "tailwindcss";`
- 그 외 커스텀 CSS는 두지 않는다.
- 폰트는 외부 `@import` 없이 Tailwind arbitrary font family 또는 system fallback으로 처리한다.
- `DESIGN.md`의 시각 언어는 TSX의 Tailwind class 조합으로 구현한다.

## 하위 컴포넌트 수정 기준
- 콘텐츠 책임은 유지한다.
- 새 디자인 표현에 필요한 Tailwind class/래퍼를 직접 추가한다.
- 새 공용 컴포넌트는 만들지 않는다.
- Hono JSX이므로 `className`이 아니라 `class`만 사용한다.
- 동작 계약이 있는 요소는 유지한다.
  - `.hero-preview`
  - `.site-footer`
  - `data-copy-target="llm-install-markdown"`
  - `id="llm-install-markdown"`
  - clipboard inline script

## 테스트 보정 기준
- 유지할 검증
  - 루트 렌더링 성공
  - 핵심 콘텐츠 문자열
  - 설치 명령/복사 UI/clipboard script
  - Open Graph/meta 태그
- 바꿀 검증
  - 예전 green/light CSS 문자열 중심 검증은 제거한다.
  - `src/style.css`는 `@import "tailwindcss";`만 검증한다.
  - 새 디자인은 HTML에 포함된 대표 Tailwind 클래스와 주요 콘텐츠로 검증한다.

## 검증 명령

2. `pnpm typecheck`
3. `pnpm test`
4. `pnpm build`
5. 가능하면 `pnpm dev` 후 `/` 페이지를 브라우저/HTTP로 확인

## 리스크와 완화책
- 폰트 외부 로드 실패
  - 외부 폰트 로드를 쓰지 않고 system fallback/arbitrary font-family로 처리한다.
- Tailwind 유틸리티 클래스가 지나치게 길어짐
  - 반복되는 class 문자열은 컴포넌트 내부 상수로 정리하되, 과도한 abstraction은 피한다.
- 테스트가 예전 스타일에 묶여 실패
  - 디자인 세부가 아니라 새 계약 중심으로 assertion을 보정한다.
- 모바일에서 장식 레이어가 과함
  - Tailwind responsive prefix로 grid/overlay/spacing을 단순화한다.
